Vintagen

Objective

This is an era for online shopping and this app is designed to make their online shopping experience efficient and convenience and increase the profitability of the seller.

The objective of this project is to design the UI for the mobile version of a vintage clothing sales app which will allow users to be able to find the items that they want and also allow them to sell off the items that they have.

Who

>

Primarily 25 to 34-year-olds who want to buy and sell vintage clothing online.

Key Functionalities

  • Create a profile
  • Upload items to sell
  • Leave reviews for other sellers

User Flow

User flows are the first stage of wireframing, normally based on user stories. Since the client wants 3 functionalities, I have created a simple user flow to let me visualize all the different routes available for a user to take and which screens a user sees as they go through the ‘flow’.

Low Fidelity

In this low-fidelity wireframes step, I brainstorm the layout of the content of each screen. They aren’t detailed and are concerned with loosely defining where everything will go. For example, will users find the navigation at the top or will it be a sidebar?

Problem: Creating a low fidelity prototype offers numerous advantages including time-saving. However since there are a lot of different layout possibilities, it is challenging to get the right choice for everyone all of the time.

Solution: Even if the prototype is in paper sketch form, it is important to get perspective from users. This will save time and get in the right direction start from the beginning.

Mid Fidelity

Once I am happy with the low-fidelity wireframes and how the overall structure of the content is looking, I refine and adjust the layout. Not only that I also start thinking about grid systems and visual hierarchy, too.

Problem: This is the phase where color and typography are matter. Depend on the user demographic, choosing the right color palette and typography is important.

Solution: This is where the result of competitor analysis comes in handy. Based on what other companies of the same field are using and based on the age range of the target users, I picked the color palette which represents youth colors (summer and spring colors) and easy-to-read typography.

High Fidelity

Although a low/mid-fidelity prototype can be an excellent way to quickly test ideas, once a project is more advanced and have a clearer idea of the final goals, a high fidelity prototype is a better way to visualize the final product design.

Problem: There should be no difference between this high fidelity stage vs the final product. This will provides testers with a more accurate indication of user experience. This should be the stage where user testing can’t be voided.

Solution: It is important to do at least 2 user testing at this stage and made final adjustments based on the feedback.
